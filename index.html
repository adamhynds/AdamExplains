Lesson 1: Make a Basic Website and Host for Free Using GitHub Pages

Starting your journey in web development? This lesson will guide you through the process of creating a basic website and hosting it for free using GitHub Pages. Perfect for beginners!

1. Create a GitHub Account

If you don't already have a GitHub account, create one at GitHub's website.

2. Create a New Repository

  • Once logged in, click on the "+" icon in the top right corner and select "New repository".
  • Name your repository. For a user or organization site, name it [your-username].github.io. For project sites, you can choose any name.
  • Initialize the repository with a README (optional, but recommended).
  • Click "Create repository".

3. Prepare Your Website

  • If you already have a website, ensure it's in a folder on your computer. The main HTML file should be named index.html.
  • If you don’t have a website, create a simple HTML file named index.html and write some basic HTML code.

4. Upload Your Website

  • Open your repository on GitHub.
  • You can upload your files by dragging and dropping them into the repository, or by using the "Add file" button and selecting "Upload files".
  • Commit the changes (you can enter a commit message to describe what you are uploading).

5. Enable GitHub Pages

  • Go to your repository settings (click "Settings" on the repository toolbar).
  • Scroll down to the "GitHub Pages" section.
  • Under "Source", select the branch where your website is stored (usually main).
  • Click "Save".

6. Customization and Configuration

  • (Optional) Choose a theme or customize your index.html.
  • (Optional) Add additional files like CSS, JavaScript, images, etc.
  • (Optional) For a project site, you can set a custom path in the settings.
  • (Optional) Add a custom domain if you have one.

7. Accessing Your Website

  • Once GitHub Pages is enabled, your site will be publicly accessible.
  • For user/organization sites, the URL will be https://[your-username].github.io.
  • For project sites, the URL will be https://[your-username].github.io/[repository-name].

8. Update Your Website

  • You can update your site anytime by uploading new files or modifying existing ones directly on GitHub.
  • Each time you commit changes, GitHub Pages will automatically rebuild your site.

9. Check Your Email

GitHub might send you an email if your site has been published or if there are issues with the publishing process.
